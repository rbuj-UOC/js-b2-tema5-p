# Tema 5. Técnicas para mejorar la estructura y robustez de código

## Unidad 1. Gestión de errores y excepciones

## Ejercicio 5.1.1 (Castellano)

En este ejercicio vamos a trabajar con la generación de errores y excepciones, aunque ya los hayamos visto de pasada al trabajar con clases en el caso de las clases abstractas, por ejemplo.

Para ello vamos a realizar un ejercicio partiendo de un inventario que tienes en el código de partida del ejercicio.
Este inventario contiene una serie de productos en un array de objetos, cada uno de los cuales representa un producto y tienen la siguiente estructura común:

```js
{
    code: 156,
    name: 'Black',
    price: 88.99,
    discount: 9,
    amount: 50
},
```

Las características de este inventario es que:

- No puede tener productos con el mismo `code`.
- No puede tener productos con el mismo `name`.
- No puede tener productos con un precio inferior a `50`.
- No puede tener descuentos inferiores a `0` o superiores a `10`.
- No puede tener una cantidad negativa.

### A. Clase `IventoryManager`

Se ha de implementar una clase `InventoryManager` que ha de contar con:

- Una propiedad privada `productList` que contendrá la lista de productos.

- Un método `init` que recibe como único parámetro `products` que contiene la lista de productos inicial.

- Un método `validateProduct` que recibe como parámetro un producto y valida que cumpla todos los requisitos indicados anteriormente.

- Un método `addProduct` que recibe un producto y valide que disponga de todos los datos y cumpla los requisitos anteriores antes de ser añadido a la `productList`

Adicionalmente:

### B. Método `init`

Este método es el encargado de importar (agregando cada uno mediante el método `addProduct`) los diferentes productos que reciba en el parámetro `products` a la `productList`.

Además ha de gestionar los problemas que puedan producirse en ese proceso, de tal forma que al final de toda la importación:

- Devuelva `true` si todos los productos se importaron correctamente sin problemas.

- Devuelva una `array` con la lista de los problemas encontrados por cada producto problemático.

### C. Método `nProducts`

Este método simplemente devolverá la cantidad de productos contenidos en la `productList`.

### D. Método `validateProduct`

Este método es el encargado de validar un producto para decidir si se añade o no a la `productList`.

> 💡 **NOTA.** Para ello ha de implementar la siguiente funcionalidad (ten en cuenta que puedes dividir la funcionalidad requerida en otros métodos privados de la clase para organizar mejor el código si lo deseas):

La forma y orden en la que se debe llevar a cabo la validación es la siguiente:

- Si alguna de las propiedades está vacía o no es del tipo requerido (`code`, `discount` y `amount` un número entero no negativo, `name` un **string** no vacío y `price` un número positivo flotante con máximo 2 decimales) lanza el siguiente error:

```
ERROR_DATA. Alguno de los datos del producto ([code]) no tiene un formato válido.
```

- Si el código del producto a añadir ya se encuentra en la lista de productos, lanzará el error siguiente:

```
INVENTORY_CODE. Ya existe otro producto con ese código ([code]).
```

- Si la lista de productos ya cuenta con un producto que tenga el mismo nombre del producto a añadir, lanzará el siguiente error:

```
INVENTORY_NAME. El nombre del producto ([code]) ya existe.
```

- Si el producto a añadir tiene un precio inferior a 50, lanza el error siguiente:

```
INVENTORY_PRICE. El precio del producto ([code]) no puede ser inferior a 50.
```

- Si el producto a añadir tiene un descuento que no se encuentre entre 0 y 10, lanzará el siguiente error:

```
INVENTORY_DISCOUNT. El descuento del producto ([code]) debe estar entre 0 y 10.
```

- Si el producto a añadir tiene una cantidad negativa, genere el error siguiente:

```
INVENTORY_AMOUNT. La cantidad de producto ([code]) no puede ser inferior a 10.
```

- Si por el contrario, el producto es totalmente válido el método retorna `true`.

### E. Método `addProduct`

Este método se encarga de añadir un producto al inventario, por lo que recibe como único parámetro `product` que es un objeto con la estructura ya comentada, implementando la siguiente funcionalidad:

- Valida el producto mediante el método `validateProduct`.

- Si es válido, lo añade a la `productList`.

- Si no es válido, recoge el error generado por `validateProduct`.

- Tanto si es válido como si no (uso de `finally`) retorna un objeto con esta estructura:

```js
{
    status: [true | false]
    message: [mensaje_resultado]
}
```

Donde:

- `status`: [true | false]: representa un **booleano** en función de si se pudo añadir el producto o no.

- `message`: [mensaje_resultado]: es un **string** que en caso de error contiene el error recibido por el método `validateProduct` y en caso de éxito contendrá la cadena `INVENTORY_ADD. El producto ([code]) ha sido añadido con éxito al inventario.`

> 💡 NOTA: Ten en cuenta que siempre que veas `[code]` ha de ser sustituido por el valor del código del producto en cuestión.<br>

Por ejemplo, para un producto con código `152` algunos mensajes serían:

```
INVENTORY_NAME. El nombre del producto (152) ya existe.

INVENTORY_AMOUNT. La cantidad de producto (152) no puede ser negativa.

INVENTORY_ADD. El producto (152) ha sido añadido con éxito al inventario.
```

### F. Exportación de clase

Exporta la clase `IventoryManager` para poder ser usada en otros módulos.

# Tema 5. Tècniques per millorar l'estructura i la robustesa de codi

## Unitat 1. Gestió d'errors i excepcions

## Exercici 5.1.1 (Català)

En aquest exercici treballarem amb la generació d'errors i excepcions, encara que ja els hàgim vist de passada en treballar amb classes en el cas de les classes abstractes, per exemple.

Per fer-ho, farem un exercici partint d'un inventari que tens al codi de partida de l'exercici.
Aquest inventari conté una sèrie de productes en un array d'objectes, cadascun dels quals representa un producte i tenen la següent estructura comuna:

```js
{
     code: 156,
     name: 'Black',
     price: 88.99,
     discount: 9,
     amount: 50
},
```

Les característiques daquest inventari és que:

- No pot tenir productes amb el mateix `code`.
- No podeu tenir productes amb el mateix `name`.
- No pot tindre productes amb un preu inferior a `50`.
- No podeu tenir descomptes inferiors a `0` o superiors a `10`.
- No pot tindre una quantitat negativa.

### A. Classe `IventoryManager`

S'ha d'implementar una classe `InventoryManager` que ha de comptar amb:

- Una propietat privada `productList` que contindrà la llista de productes.

- Un mètode `init` que rep com a únic paràmetre `products` que conté la llista de productes inicial.

- Un mètode `validateProduct` que rep com a paràmetre un producte i valida que compleixi tots els requisits indicats anteriorment.

- Un mètode `addProduct` que rep un producte i validi que disposi de totes les dades i compleixi els requisits anteriors abans de ser afegit a la `productList`

Addicionalment:

### B. Mètode `init`

Aquest mètode és l'encarregat d'importar (afegint cadascú mitjançant el mètode `addProduct`) els diferents productes que rebi al paràmetre `products` a la `productList`.

A més, ha de gestionar els problemes que es puguin produir en aquest procés, de manera que al final de tota la importació:

- Torneu `true` si tots els productes es van importar correctament sense problemes.

- Torneu una `array` amb la llista dels problemes trobats per cada producte problemàtic.

### C. Mètode `nProducts`

Aquest mètode simplement tornarà la quantitat de productes continguts a la `productList`.

### D. Mètode `validateProduct`

Aquest mètode és l'encarregat de validar un producte per decidir si s'afegeix a la `productList` o no.

> 💡 **NOTA.** Per això ha d'implementar la funcionalitat següent (tingues en compte que pots dividir la funcionalitat requerida en altres mètodes privats de la classe per organitzar millor el codi si ho desitges):

La forma i ordre en què s'ha de dur a terme la validació és la següent:

- Si alguna de les propietats està buida o no és del tipus requerit (`code`, `discount` i `amount` un nombre enter no negatiu, `name` un **string** no buit i `price` un nombre positiu flotant amb màxim 2 decimals) llança el següent error:

```
ERROR_DATA. Alguna de les dades del producte ([code]) no té un format vàlid.
```

- Si el codi del producte a afegir ja és a la llista de productes, llançarà l'error següent:

```
INVENTORY_CODE. Ja existeix un altre producte amb aquest codi ([code]).
```

- Si la llista de productes ja compta amb un producte que tingui el mateix nom del producte a afegir, llançarà el següent error:

```
INVENTORY_NAME. El nom del producte ([code]) ja existeix.
```

- Si el producte a afegir té un preu inferior a 50, llença l'error següent:

```
INVENTORY_PRICE. El preu del producte ([code]) no pot ser inferior a 50.
```

- Si el producte a afegir té un descompte que no estigui entre 0 i 10, llançarà el següent error:

```
INVENTORY_DISCOUNT. El descompte del producte ([code]) ha d'estar entre 0 i 10.
```

- Si el producte a afegir té una quantitat negativa, genereu l'error següent:

```
INVENTORY_AMOUNT. La quantitat de producte ([code]) no pot ser inferior a 10.
```

- Si per contra, el producte és totalment vàlid el mètode retorna `true`.

### E. Mètode `addProduct`

Aquest mètode s'encarrega d'afegir un producte a l'inventari, per la qual cosa rep com a únic paràmetre `product` que és un objecte amb l'estructura ja comentada, implementant la funcionalitat següent:

- Valida el producte mitjançant el mètode `validateProduct`.

- Si és vàlid, ho afegeix a la `productList`.

- Si no és vàlid, recull l'error generat per `validateProduct`.

- Tant si és vàlid com si no (ús de `finally`) retorna un objecte amb aquesta estructura:

```js
{
     estatus: [true | false]
     message: [missatge_resultat]
}
```

On:

- `status`: [true | false]: representa un **booleà** en funció de si es va poder afegir el producte o no.

- `message`: [missatge_resultat]: és un **string** que en cas d'error conté l'error rebut pel mètode `validateProduct` i en cas d'èxit contindrà la cadena `INVENTORY_ADD. El producte ([code]) ha estat afegit amb èxit a l'inventari.`

> 💡 NOTA: Tingues en compte que sempre que vegis `[code]` ha de ser substituït pel valor del codi del producte en qüestió.<br>

Per exemple, per a un producte amb codi `152` alguns missatges serien:

```
INVENTORY_NAME. El nom del producte (152) ja existeix.

INVENTORY_AMOUNT. La quantitat de producte (152) no pot ser negativa.

INVENTORY_ADD. El producte (152) ha estat afegit amb èxit a l'inventari.
```

### F. Exportació de classe

Exporta la classe `IventoryManager` per poder ser utilitzada en altres mòduls.
